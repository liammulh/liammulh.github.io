Easier to ask for forgiveness than permission (EAFP) and look before you leap (LBYL) are different approaches for dealing with error-prone code.
Published: Wednesday, October 4, 2023
Tagged under: python, software-engineering

I was skimming the Python glossary today and noticed the entry for the EAFP [1] coding style. In the easier to ask for forgiveness than permission coding style, you assume an object has the right keys and you throw an error if it doesn't. This involves wrapping your assumption in try-except blocks. This style is common in the Python community, and it's contrasted with the LBYL [2] coding style. In the look before you leap coding style, you explicitly test for pre-conditions before doing anything potentially dangerous. This involves adding lots of conditional statements to your code. The LBYL coding style is common in C. It also seems to be common in JavaScript, based on my experience. The Python glossary notes that in a multi-threaded environment, LBYL is potentially dangerous. One thread could be checking if a key is in an object, and another thread could have already deleted the key. This can be solved by using locks or by using the EAFP style.
